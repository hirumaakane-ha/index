Night 3 – Tue, Jul 10th, ’18

Setting up a botched FTP server

  • My initial approach for the website is perhaps trying to answer the question of how to regularly update the website with local content and edit the website HTML and CSS source codes. So, I came by FTP – a file transfer protocol named ‘File Transfer Protocol’ and in my head I was imagining the incredible workflow I can construct using FTP as the ace in my hand. Right after my setting up the DNS and stuff, I dived head-first to installing and implementing FTP on my Ubuntu host server. Little did I know that I was in for my first greatest bamboozling in my short coding lifespan.
  • Till this date, I am still mundane about why my FTP setup did not work even though I followed every instruction on the AWS, DigitalOcean websites and every troubleshooting tips that I could find on Stack Exchange. The error that made it impossible for me to realize my hope of completing the FTP server was I could not connect to the server through various FTP programs that I installed because of some authentication errors.
  • I tried, of course, to lessen whatever security methods implemented on the Virtual Host via both the security group options on EC2 and the authentication protocols Ubuntu server OS itself. To no avail. At that point, I completely gave up and reset the security settings to default for fear of any future unwanted accesses to my server.
  • I decided to stick with the git repository and Github to commit changes from my local computer and to pull the changes manually by ssh-ing into the server. Below Is my current and only known workflow.
  • TBH, I don't really know what is the point or the function of an FTP server. Tee hee. :)

    ver1.0

  1. Changes from local PC (desktop/laptop, same thing)
  2. Commit and push changes from local PC
  3. SSH-ing into Ubuntu server
  4. Sudo pull the changes from the (public, *sigh*) git repo to a folder X
  5. Using command cp -a to copy every file/subfolder from folder X to /var/www/…
  6. Delete unnecessary or extra remnants

    ver2.0

  1. Changes from local PC
  2. Commit and push changes
  3. SSH-ing into server
  4. Pull changes directly to /var/www/(git repo folder)
  • While it is SUPER insecure as everything is saved on a public git and not so efficient to do that, it is the only way I know how to do it. So, please, any suggestions would be so welcomed.
